선택동기
    - 스타벅스의 메인화면에 슬라이딩을 구현하고 싶었으며 서브페이지의 형식이 비슷하여 한번에 다양한 서브페이지를 구현할 수 있을것 같다고 생각함.
    - 스타벅스의 색감이 연한 베이지와 짙은 초록색의 색감이 눈이 편함

구현기능
    모든페이지 공통기능
        - gnb메뉴
            -> html파일이 아닌 js파일에 각 항목의 데이터를 담고 for in과 for of를 통해 화면에 출력
            -> 각 항목의 데이터가 출력될 때 그 높이를 이용하여 백그라운드의 전체 높이를 결정함.
            -> 화면 스크롤 시에도 position: fixed를 이용하여 화면 내부에서 사라지지 않도록 함.
            -> gnb의 제목 클릭시 각 서브페이지로 이동, 로고 클릭시에는 첫 메인 페이지로 이동.
            -> 960px 미만의 너비화면에서는 데스크탑 gnb메뉴는 display: none; 하고 모바일 gnb화면을 보이도록 함.
            -> 모바일 화면에서는 바 버튼 클릭시 오른쪽에서 메뉴 등장하며 
            메뉴 외의 화면은 흐린 검정색으로 가려서 클릭되지 않게함.
            -> 각 메뉴 클릭시 하위 메뉴가 있을 경우 아래로 늘어나며 서브메뉴 보이도록 함.
            -> 각 서브메뉴에서 '한눈에 보기' 버튼 클릭시 서브 페이지로 이동.
        
        -footer메뉴
            -> display: flex; 를 이용해서 각 리스트를 옆으로 나열할 수 있도록 하고, 화면의 폭이 좁아지면 flex-wrap: wrap; 속성으로 이중으로 나열되게 함.
            -> width가 960px 미만일 때, 메뉴를 세로로 나열함.
            -> 세로로 나열된 상태에서 각 항목 클릭시 서브메뉴가 아래로 밀어내며 등장함.
    - 메인페이지
        -> 첫화면 배경을 setInterval 함수로 2초마다 슬라이딩 하도록 설정
        -> 우측 광고를 fixed하여 스크롤 시에도 위치가 고정된 상태로 이동.
        -> 프로모션 화면 버튼 클릭시 height값을 내부 박스 높이만큼 늘려 보이도록 설정
        -> 프로모션 리스트 3초마다 무한 슬라이딩 기능, 화살표 버튼 클릭시 그 방향으로 넘어가게 하고, 넘어가는 순간에 clearInterval로 자동 슬라이딩 함수 멈추고 2초후에 다시 무한 슬라이딩 기능 실행
        -> 페이지 스크롤에 따라 클래스를 넣고 빼면서 각 이미지와 설명이 양쪽에서 등장하는 애니메이션, opacity를 점점 증가시키며 선명하게 하는 애니메이션 구현
    
    - 서브 페이지 공통
     -> 전체 페이지 속박스의 넓이를 1100px로 고정한 후 그 안에 요소 배치
     -> 각 페이지마다 제일 위에 페이지 제목을 h2태그 안에 작성
     -> 메인 배경이 있는 경우 제목 밑에 위치하며, 배경이 없을 경우 보도자료 박스가 위치.
     -> 미디어쿼리를 이용하여 화면의 너비가 960px미만으로 줄어들 경우 배경사진을 모바일용으로 변경하여 높이가 높은 사진으로 변경함.
     -> 보도자료 박스는 리스트를 display: flex;, flex-direction: column; 으로 세로로 나열한 후 setInterval함수를 통해 아래에서 위로 각 항목이 밀고 올라오는 애니메이션을 구현.
     -> 보도자료 박스와 웹사이트 이용안내 박스는 너비 660px미만에서는 세로로 배치되어 위 아래에 위치함. 세로로 배치되면 보도자료 박스의 '보도자료' 텍스트는 사라짐.
     -> 각 서브페이지별 사진, 텍스트, 자세히보기버튼을 변경하며 미디어쿼리를 통해 너비가 줄어들면 텍스트와 자세히보기버튼의 크기와 위치를 줄어들도록 함.

     - COFFEE 페이지
        -> 배경이미지를 setInterval로 2초마다 opacity를 0에서 1로 변경하며 두 이미지를 교차해서 보이도록 함.
        -> 화살표 버튼 클릭 시 다음 페이지의 opacity를 1로 만들면서 setInterval함수를 멈추고 2초후 재실행.
    
    - MENU 페이지
        -> 데이터로 담아둔 제품 이름과 사진을 js로 출력함.
        -> '관련제품보기' 버튼 클릭시 총 리스트의 높이를 구해 총 박스의 높이를 결정해서 늘어나도록 설정.
        -> 각 항목들은 display: flex; 로 나열하였으며 첫 width는 1100px이지만 양쪽 너비가 모바일 화면으로 줄어들면 width를 auto로 설정하여 리스트의 column 갯수를 줄어들도록 함.

    - CORPORATE 페이지
        -> 다른 서브페이지와 다르게 서브제목에 배경사진을 넣음
        -> '상품 안내 및 구매 절차' 박스를 클릭하면 상품과 구매절차가 나오며, '결제 수단 안내' 박스를 클릭하면 결제 방법에 대한 내용으로 콘텐츠 박스가 바뀜.
        -> 상품안내에 해당하는 버튼을 클릭하면 각 상품의 구매 절차 이미지와 텍스트가 바뀌며,  각 항목의 유의사항 텍스트 역시 바뀜.
        -> 너비 960px미만에서는 각 항목의 구매절차 이미지가 세로 형식으로 바뀌며 텍스트가 각 이미지의 오른쪽에 배치됨.
        -> 너비 730px미만에서는 제품들의 버튼이 select로 바뀌며 선택시 각 항목의 절차가 나타남.
        -> 넓은 화면에서의 제품들의 버튼과 730px 미만의 select 목록은 서로 연동되어 있으며 어느 하나를 바꿔도 다른 화면에서 유지됨.
        -> 660px미만에서는 '상품 안내 및 구매 절차' 박스와 '결제 수단 안내' 박스가 세로로 배치되며 width가 100%로 바뀜
